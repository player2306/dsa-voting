Introduction

  • Finds a majority element in a list using counters.
  • Covered in class: majority elements and efficient tracking.
  • We used ChatGPT to clarify concepts and organize our discussion.

Interaction 1 — Definition

Q: What is Moore's Voting Algorithm and what problem does it solve?

A: Linear-time, constant-space method to find a majority element (> n/2) by canceling out different elements.

Accurate and concise.

Interaction 2 — Step by Step

Q: Can you explain how it works step by step?

Example: [2, 2, 1, 1, 2] candidate = null, count = 0 see 2 -> candidate = 2, count = 1 see 2 -> count = 2 see 1 -> count = 1 see 1 -> count = 0 see 2 -> candidate = 2, count = 1

Clear illustration of cancellation.

Interaction 3 — Complexity

Q: Time and space complexity and why optimal?

A: Time O(n), Space O(1); any solution must inspect all elements.

Matches theory and our implementation.

Interaction 4 — Applications

Q: Real-world applications?

A: Live polls, streaming vote counts, sensor consensus, network analysis, game input smoothing.

Relevant and practical.

Interaction 5 — Pros, Cons, Variants

Pros

  • O(n) time, O(1) space
  • Single pass
  • Streaming friendly

Cons

  • Only strict majority
  • Requires verification
  • No frequency counts

Variant: Track two candidates to find elements > n/3, then verify.

Interaction 6 — Edge Cases + Comparison

Edge cases

  • Empty array
  • Single element
  • No majority
  • All identical
  • Alternating values

Comparison

  • Moore's: O(n), streaming
  • Sorting: O(n log n), needs all data

Correct and focused, with minor gaps on verification emphasis.

Application Example — Online Poll

  • Votes arrive as a stream; storing all votes is impractical.
  • Maintain candidate + count in one pass.
  • Verify the candidate at the end.
  • Fits O(n) time and O(1) memory requirements.

Benchmark Results

  • Growing array up to 1,000,000 elements.
  • Moore's: roughly linear, ~0.03 us at 1 to ~709 us at 1,000,000.
  • Sort-based: O(n log n), ~28 ms at 1,000,000.
  • Brute force: O(n^2), impractical after 50,000.

Reflection

  • Strength: Fast, clear explanations and example-driven learning.
  • Weakness: Candidate must be verified when majority is not guaranteed.
  • Prompting mattered: sharper questions improved depth.

Improvements

Conclusion
